Proposed Microservices Architecture

Distributed system with REST and Kafka communication

System Diagram

Choose Your Preferred Communication Strategy

A

Synchronous Orchestration (Recommended)

Payment Service calls Wallet via REST (blocking) to ensure immediate balance check, then sends to Ledger via Kafka (async). Best for consistency where it matters.

B

Fully Event-Driven (Choreography)

All services communicate via Kafka topics. Payment emits 'Created', Wallet reacts and emits 'Debited', etc. Highly decoupled but harder to track state across services.

Project Structure Details

  • /payments-common: Shared DTOs and Logic
  • /payment-service: Core State Machine
  • /wallet-service: Account balances and debits
  • /ledger-service: Transaction history and audit
